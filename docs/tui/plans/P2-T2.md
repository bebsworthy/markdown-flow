## 2. `ink-testing-library@4` API facts

- `render(tree)` returns `{ rerender, unmount, cleanup, stdout, stderr, stdin, frames, lastFrame }`.
- `lastFrame(): string | undefined` ‚Äî composed string of last Ink render; plain text (no ANSI emitted by the library).
- `stdin.isTTY = true`; `setRawMode()` is a no-op stub ‚Üí Ink's raw-mode gate passes.
- `stdin.write(data)` is **synchronous**: emits `'readable'` then `'data'` same tick. No `await` or microtask flush needed for single keystrokes.
- `stdout.columns = 100` (not relevant for the scaffold).

## 3. `tsconfig.json` modification

**Decision: remove `"test"` from `exclude`, add `"test"` to `include`. Do NOT introduce a separate `tsconfig.test.json`.**

Final shape:

```json
{
  "extends": "../../tsconfig.base.json",
  "compilerOptions": {
    "rootDir": "src",
    "outDir": "dist",
    "declaration": false,
    "sourceMap": true
  },
  "include": ["src", "test"],
  "exclude": ["node_modules", "dist", "vhs"]
}
```

**Why.**

- `npm run lint -w packages/markflow-tui` = `tsc --noEmit`. Excluding `test` would let bad imports / wrong types in tests escape lint and only surface at test-run time.
- Second tsconfig (`tsconfig.test.json`) adds file count + requires editing the lint script. Over-engineering for two test files.
- `rootDir: "src"` + `include: ["src", "test"]` mismatch is harmless with `noEmit`. `tsup` drives the actual build from `tsup.config.ts` and ignores tsconfig's `include`.

## 6. `test/components/scaffold.test.tsx` (new)

```tsx
import React from "react";
import { describe, it, expect, vi } from "vitest";
import { render } from "ink-testing-library";
import { App } from "../../src/app.js";

describe("scaffold App", () => {
  it("renders the scaffold text", () => {
    const { lastFrame } = render(<App onQuit={() => {}} />);
    expect(lastFrame()).toContain("markflow-tui ¬∑ scaffold");
  });

  it("calls onQuit when q is pressed", () => {
    const onQuit = vi.fn();
    const { stdin } = render(<App onQuit={onQuit} />);
    stdin.write("q");
    expect(onQuit).toHaveBeenCalledTimes(1);
  });
});
```

**Notes.**

- Import path `"../../src/app.js"` ‚Äî `.js` on `.tsx` source required by `moduleResolution: "bundler"`.
- Middot is U+00B7 (`¬∑`). **Copy from `src/app.tsx`** rather than retyping ‚Äî `toContain` is a byte-level comparison.
- `stdin.write("q")` synchronous; no `await`.
- `React` import kept for parity with `src/app.tsx`; `jsx: "react-jsx"` makes it technically unused but harmless.

```markdown
# markflow-tui test harness

Two layers of the [5-layer TUI testing stack](../../../docs/tui/testing.md#6-recommended-stack-for-markflow) live here:

- **Layer 1 ‚Äî Unit (vitest).** Pure functions with no Ink. Example: `test/reducer.placeholder.test.ts`.
- **Layer 2 ‚Äî Component (ink-testing-library).** React components rendered through Ink's in-memory stdout. Example: `test/components/scaffold.test.tsx`. Assert on `lastFrame()` output; drive keypresses via `stdin.write(...)`.

Higher layers (3 node-pty, 4 `@microsoft/tui-test`, 5 VHS) arrive in Phase 9 and live elsewhere.

## Commands

Run from the repo root:

    npm test -w packages/markflow-tui             # one-shot run (CI-friendly)
    npm test -w packages/markflow-tui -- --run    # explicit --run flag, equivalent
    npm run test:watch -w packages/markflow-tui   # watch mode during development
    npm run lint -w packages/markflow-tui         # tsc --noEmit, now also checks test/

## Conventions

- Import `describe`, `it`, `expect`, `vi` explicitly from `"vitest"` ‚Äî globals disabled.
- Use `.js` suffix when importing from `src/` (required by `moduleResolution: "bundler"`).
- Do not add inline ANSI snapshots; use `toContain`/`toMatch` on canonicalized strings.
```
